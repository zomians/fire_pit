# ADR-0001 — 図面ソースを React/JSX から vanilla JS パラメトリックへ

## 文脈
旧成果物（`reference/legacy/`）は claude design 製で、React 18 + Babel standalone を CDN から読み、`*.jsx` をブラウザで実行時トランスパイルして SVG を描く構成だった。本リポジトリは「土台から再現可能に」作り直すのが目的で、成果物は外部業者へ渡す印刷図面。姉妹案件 hana_block は「React なし・自己完結 HTML+SVG・ビルドなし」を採用している。

## 決定
図面ソースは **vanilla JS（依存ゼロ・パラメトリック）** とする。React / Babel / CDN を使わない。六角形 R・焚き火半径などを入力に、頂点・18区画・焚き火部を計算で導く幾何（旧 `hexagonGeometry` 等）は vanilla 化して `lib/firepit.js` に移植し、各シートが相対 `<script src>` で読む。外部依存はフォント（Google Fonts）のみ。

## 理由
- **印刷の堅牢性**: CDN（React/Babel/unpkg）依存は network・バージョン寿命・実行時トランスパイルに左右され、`file://` で開いて印刷する用途に脆い。vanilla は依存ゼロ。
- **再現性**: 再現の源を claude design のバンドラ／固定版 React/Babel に委ねたくない。vanilla + docs なら源が自己完結。
- **パラメトリックは残す**: 焚き火台の幾何は18区画で反復的。手作図よりコードで導く方が正確・改変容易。「React の脆さ」と「パラメトリックの利点」は分離でき、後者だけ残せる。
- **姉妹案件との整合**: hana_block と技術哲学を揃え、宿設備プロジェクト群で頭の使い方を1つに保つ（CDN/ビルドなし）。ただし共有 `lib/` を持つ点だけは hana_block の「1ファイル完全自己完結」から逸脱（→ ARCHITECTURE）。

## トレードオフ / 却下案
- **React/JSX 維持**: コンポーネント再利用・対話的キャンバス（旧 design-canvas）・claude design 往復が利点だが、いずれも「設計探索 phase」の価値で、印刷成果物＋再現性という目的に乗らない。却下。
- **standalone バンドルのみ保持**: 1.3MB のブラックボックスで人が編集できず、再現の主体になれない。却下。
- **オフライン生成スクリプト（C2）**: 出力は最堅牢だがビルドツールチェーンが増える。現規模ではオーバースペック。将来の再検討点として保留。
